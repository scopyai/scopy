## Find real bugs without the noise

Scopy goes beyond the changed lines. It explores the repository around a change, builds the context needed to reason about it, and independently verifies candidate issues before leaving focused feedback.

|                            |                                                                                                                    |
| -------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| **Context-aware reviews**  | Understands dependencies and surrounding code before raising an issue.                                             |
| **Natural-language rules** | Turns your team's standards into checks without another configuration language.                                    |
| **Flexible controls**      | Choose what Scopy watches and when automatic reviews should run.                                                   |
| **Workspace analytics**    | See review activity, issue severity, languages and frequently flagged files to know where to be careful next time. |
| **Use anywhere**           | Use Scopy Cloud for the quickest setup or run the open-source stack yourself.                                      |

## Get started

The fastest route is [Scopy Cloud](https://app.scopy.dev): connect GitHub, select your repositories, and Scopy will review the next pull request that matches your settings.

Prefer to own the deployment? Follow the [self-hosting guide](https://docs.scopy.dev/self-hosting/overview) for local setup, configuration, and architecture details on how Scopy actually works.

## Contributing

Issues and pull requests are welcome. If you have an idea, found a false positive, or want to improve a review adapter, [open an issue](https://github.com/scopyai/scopy/issues) and tell us what you observed.
