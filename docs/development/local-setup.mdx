## Workspace scripts

Run commands from the repository root unless noted otherwise.

| Command | What it does |
| --- | --- |
| `pnpm dev` | Runs development tasks through Turbo. |
| `pnpm build` | Builds all workspaces through Turbo. |
| `pnpm typecheck` | Runs TypeScript checks across workspaces. |
| `pnpm lint` | Runs lint tasks where configured. |
| `pnpm format` | Runs formatting tasks where configured. |

## API development

The API workspace is `apps/api`.

| Command | What it does |
| --- | --- |
| `pnpm --filter api dev` | Starts the Elysia API and Graphile Worker together. |
| `pnpm --filter api dev:api` | Starts only the Elysia API on `PORT`, default `3001`. |
| `pnpm --filter api worker` | Starts only the worker process. |
| `pnpm --filter api db:generate` | Generates Drizzle migrations. |
| `pnpm --filter api db:migrate` | Applies Drizzle migrations. |
| `pnpm --filter api db:studio` | Opens Drizzle Studio. |
| `pnpm --filter api worker:migrate` | Creates or updates the Graphile Worker schema. |

The API exports its Elysia app type from the `api` workspace package:

```ts
import type { App } from "api"
```

Frontend code should use that package export instead of importing from API source paths.

## Web development

The product web app is `apps/web`.

| Command | What it does |
| --- | --- |
| `pnpm --filter web dev` | Starts the TanStack Start app on `3000`. |
| `pnpm --filter web build` | Builds the app. |
| `pnpm --filter web start` | Runs the built Nitro server. |
| `pnpm --filter web typecheck` | Runs TypeScript checks. |

The web app creates the Eden treaty client in `apps/web/src/lib/api.ts`:

```ts
import { treaty } from "@elysiajs/eden"
import type { App } from "api"

export const api = treaty<App>(env.VITE_API_BASE_URL, {
  fetch: { credentials: "include" },
})
```

Use this client from React Query hooks rather than using raw `fetch` or `axios` in components.

## Shared UI

Shared UI components live in `packages/ui` and are imported through the package export:

```tsx
import { Button } from "@workspace/ui/components/button"
```

New shadcn/ui components should be added from the repository root:

```bash
pnpm dlx shadcn@latest add button -c apps/web
```

The current component setup places shared components in `packages/ui/src/components`.

## Local Docker compose

`docker-compose.local.yml` defines:

- `db`: PostgreSQL 16 on port `5432`.
- `api-migrate`: applies Drizzle and Graphile Worker migrations.
- `api`: production-like API container on port `3001`.
- `api-worker`: production-like worker container.
- `web`: production-like web container on port `3000`.

Use the full compose stack when you want to test container behavior. Use only `db` plus local TypeScript processes for faster feature development.
