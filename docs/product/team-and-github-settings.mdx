The team management page is scoped to the selected workspace.

## Members

Workspace members can view and work with the connected workspace based on their role. The current roles are:

| Role | Purpose |
| --- | --- |
| Owner | Full workspace ownership. |
| Admin | Workspace administration. |
| Member | Standard workspace access. |

The product UI shows member management for the selected workspace and current user role.

## GitHub configuration

Use **Configure on GitHub** to manage repository access and permissions for the GitHub App installation.

Use this when:

- You need to grant Scopy access to more repositories.
- You need to remove repository access.
- The GitHub App permissions need to be inspected.

If the installation was deleted or needs to be repaired, Scopy shows a reinstall action instead.

## Leaving a workspace

Non-owner users can leave a workspace from team management. Leaving removes access to that workspace, its repositories, reviews, and analytics.

Owners should transfer ownership or adjust workspace membership before attempting to leave operational workspaces.
