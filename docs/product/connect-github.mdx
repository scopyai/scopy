The first product step is connecting GitHub. Review uses a GitHub App installation to access repository metadata, receive webhook events, and publish pull request feedback.

## Onboarding steps

1. Sign in to Review.
2. Choose **Connect your GitHub repositories**.
3. Install the GitHub App on a user account or organization.
4. Select the repositories GitHub should expose to the app.
5. Return to Review.
6. Continue to repository selection.

After GitHub is connected, Review creates or updates a workspace for the installation.

## Choosing GitHub repository access

GitHub controls which repositories are visible to the app installation. Review can only show and review repositories that the GitHub App can access.

If a repository is missing in Review:

- Check whether the GitHub App is installed on the correct account or organization.
- Check whether the repository was selected in GitHub installation settings.
- Use the repository sync action in Review after changing GitHub access.

## Reinstalling or reconfiguring GitHub

Workspace owners and admins can open GitHub configuration from the team management area. Depending on workspace state, Review either opens the existing GitHub installation settings or sends you through the reinstall flow.

Use this when:

- The GitHub App installation was removed.
- New repositories need to be granted to the app.
- Organization permissions changed.
- Review shows that repository access was removed.
